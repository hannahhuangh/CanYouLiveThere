This project compares ZIP-code rent with local income and a user-selected annual income. Instead of only asking where rent is expensive, it asks where rent becomes unaffordable relative to income.

How affordability is calculated

Monthly rent is compared to monthly income using the 30% affordability rule. If rent takes 30% or less of income, it is considered affordable. This project also calculates the annual income required to afford each ZIP code’s rent: monthly rent × 12 ÷ 0.30.

Project Writeup

So far, we have built the foundation of our interactive affordability visualization project and connected multiple datasets together. We cleaned and merged Zillow rent data with IRS income data at the ZIP-code level to calculate affordability based on the 30% rent rule. We also created an interactive U.S. map that allows users to explore affordability across different regions using a salary slider. Recently, we improved the visualization by adding zoom interactions so users can zoom from the national state-level view into county-level affordability views within each state. However, there are still several features from our proposal that we have not completed yet, including affordability rankings and commute or location preference filtering.


The most challenging part of our project will likely be implementing the remaining linked visualizations and making all of the interactions work smoothly together. Our proposal includes multiple coordinated views, which all need to update dynamically based on the selected salary and location. Designing these interactions is difficult because each visualization depends on multiple datasets and filters being synchronized at the same time. Another challenge will be handling large geographic and time-series datasets efficiently while keeping the website responsive and easy to use. We also expect the storytelling and interface design to be challenging because we want users to clearly understand affordability differences without overwhelming them with too much information at once.
